Stage 1: Data Ingestion Sources

The pipeline begins by tapping into multiple data streams. We don't just pull from one source; we synthesize real-time events, historical blocks, and crucial off-chain context.

Stage 2: The Medallion Architecture Process

We don't use raw data for ML. We refine it through a 3-step process, turning immutable, uncleaned data into validated, feature-rich tables. This ensures data quality and reliability.

BRONZE
Raw Ingestion

Full blocks, raw transactions, and un-decoded logs are dumped here. Immutable, uncleaned, and schema-on-read.

‚ñº
SILVER
Cleaned & Normalized

Raw data is decoded using Smart Contract ABIs. Transactions are filtered, cleaned, and structured into validated tables.

‚ñº
GOLD
Feature Store Ready

Silver data is aggregated into time-series and behavioral features (e.g., TVL, user frequency). Ready for ML.

Stage 3: Feature Engineering (Gold Layer)

This is where data becomes intelligence. We aggregate data into key categories that our models use to find patterns. Financial and temporal features are currently the most critical drivers for prediction models.

Temporal Feature Example

By aggregating data into time-series, we can track network health and user behavior, like the average gas price, which directly impacts user activity and model predictions.

Stage 4: Consumption & ML Use Cases

The final "Gold" data feeds a variety of ML models. Our pipeline is built to be highly effective for tasks involving anomaly detection and time-series price prediction.
